# v13.22.0 - Formula Validation & Feature System Enhancements

**Release Date**: March 2026

## Major Features

### Regex Formula Validation for Roll Formulas

A new validation system for roll formulas that ensures formula correctness before action card execution.

**What is Regex Formula Validation?**

This feature adds regular expression-based validation to roll formulas used in action cards, preventing invalid formulas from causing execution errors. The system validates formulas against expected patterns before they're used in gameplay.

**Key Capabilities**

- **Regex Field Validation**: New regex field in action card data model for formula pattern matching
- **Formula Validator Class**: New [`FormulaValidator`](../../module/services/formula-validator.mjs) service for consistent formula validation
- **Settings Integration**: System-wide settings for configuring regex patterns
- **Translation Support**: New translation keys for validation messages and error feedback

**Benefits**

- Catches formula errors before they cause gameplay disruptions
- Provides clear feedback when formulas don't match expected patterns
- Improves action card reliability and user confidence
- Supports complex formula validation requirements

**Implementation Details**

- Validation occurs during action card configuration and execution
- Regex patterns are configurable in system settings
- Validation messages are localized for international users
- Integrates seamlessly with existing action card workflow

### New Action Card Activation Methods

Enhanced action card execution capabilities with additional activation modes.

**Enhanced Execution Options**

- **Additional Modes**: New activation methods expand action card versatility
- **Flexible Execution**: More control over how and when action cards activate
- **Improved Workflow**: Better support for varied combat and roleplay scenarios

**Use Cases**

- Complex combat maneuvers requiring specific activation conditions
- Multi-stage abilities with different activation triggers
- Conditional effects based on game state
- Specialized action types for unique character abilities

### Active/Inactive Feature System

Features can now be marked as active or inactive, with automatic state management for active effects.

**What are Active/Inactive Features?**

This system allows features to have an active state that controls whether they contribute their effects to the character. Inactive features are tracked but don't apply their bonuses or abilities, with clear chat card communication when inactive features are rolled.

**Key Capabilities**

- **Active Field**: New schema field for features to track active/inactive state
- **State Change Management**: Automatic handling of active effects when feature state changes
- **Chat Card Integration**: Clear visual indication when inactive features are rolled
- **Tab Organization**: New UI tabs for managing active and inactive features
- **Translation Support**: New keys for active/inactive feature UI elements

**Benefits**

- Players can temporarily disable features without deleting them
- GMs can see when inactive features are being used
- Clear communication prevents confusion about character capabilities
- Supports temporary bonuses, conditional abilities, and equipment-based features

**UI Integration**

- Features organized into Active and Inactive tabs on character sheets
- Visual indicators show feature state at a glance
- Easy toggle between active and inactive states
- Chat cards clearly indicate when inactive features are rolled

## Bug Fixes

### Restore Target Macro Fix

**Issue**: The restore target macro was not working correctly because status effect removal was happening too late in the restoration process.

**Fix**: Moved status effect removal to the beginning of the restoration process.

**Impact**: The restore target macro now functions correctly, properly removing status effects before applying restoration effects.

**Pull Request**: #154

## Getting Started

### For Players

**Using Formula Validation**

- When creating action cards, the system will validate your roll formulas
- If a formula doesn't match the expected pattern, you'll see a clear error message
- Check your action card configuration if validation fails
- Contact your GM if you believe a formula should be valid

**Managing Active/Inactive Features**

- View your features organized into Active and Inactive tabs
- Toggle features between states as needed
- When you roll an inactive feature, the chat card will clearly indicate it's inactive
- Use inactive features for temporary bonuses or conditional abilities

**Using New Activation Methods**

- Explore the new activation modes when configuring action cards
- Choose the activation method that best fits your ability's intended use
- Test new activation methods in safe situations before using them in critical encounters

### For Game Masters

**Configuring Formula Validation**

- Access System Settings to configure regex patterns for formula validation
- Customize patterns to match your game system's requirements
- Review validation messages to ensure they're clear for your players
- Adjust validation strictness based on your campaign's needs

**Managing Feature States**

- Help players understand when to use active vs inactive features
- Review chat cards to see when inactive features are being rolled
- Use inactive features for temporary bonuses or conditional abilities
- Ensure players understand the impact of feature state on their character

**Action Card Configuration**

- Review existing action cards that may benefit from new activation methods
- Test new activation modes to understand their behavior
- Update action card documentation to reflect new capabilities
- Consider how new activation methods interact with house rules
