# v14.12.0 — Bulk Saved Damage Creator

**Release Date**: July 2026

## Major Features

### Bulk Saved Damage Creator

A new dialog for creating many saved-damage action cards at once from a folder of images. Point it at a folder in your Foundry data directory, configure the shared damage formulas and colors once, and one action card is created per image — ideal for importing a library of attack icons as ready-to-use cards.

**What's New**

- **Actor sheet entry point** — new **+Bulk** button on the Action Cards tab header (next to +Item and +Template), plus a "Bulk from Image Folder" entry in the tab's context menu.
- **Visual folder picker** — click **Browse** to open Foundry's FilePicker in folder-selection mode (`type: "folder"`), navigate to your image folder, and confirm. The active source tab (`data`, `public`, `s3`) is captured so the scan queries the right source.
- **Recursive scanning** — optional toggle descends into subfolders up to 5 levels deep. Nested images are named with their relative path (e.g. `boss/dragon.png`) so organization is preserved.
- **Intentional "noisy" names** — cards are named with the full image filename **including extension** (`dragon-breath.png`, `goblin-slash.webp`). This makes machine-generated names obvious and encourages manual review before play.
- **Shared configuration** — all cards created in a single batch share the same:
  - Resolve damage formula + type (damage/heal)
  - Power damage formula + type
  - Repetition formula (single shared count — schema only supports one)
  - Two independent "apply on every repetition" toggles (one for resolve, one for power)
  - Background and text colors
- **Group assignment** — optionally drop all created cards into a new or existing action card group, keeping the actor's action-cards tab organized after large imports.
- **Persisted configuration** — the dialog remembers the last-used folder path, source, formulas, types, toggles, colors, and recursive state via a hidden client setting. Group selection is intentionally **not** persisted (per-creation decision).
- **Pre-flight validation** — formula inputs are validated up-front via `FormulaValidator`, so an invalid formula surfaces a precise error before the atomic bulk-creation call rather than failing the whole batch opaquely.
- **Diagnostic errors** — when a scan returns no images, the error reports how many raw files were returned and from which source, making it easy to distinguish "wrong source" from "wrong path" from "wrong file types".

**Access Points**

- **+Bulk** button on the Action Cards tab header
- Right-click context menu on the Action Cards tab → "Bulk from Image Folder"
- Programmatic: `new erps.macros.BulkSavedDamageCreator({ actor }).render(true)`

**What It Doesn't Do**

Intentionally out of scope — by design — for this quick-attack-creation tool:

- **No per-image customization** — all cards share the same formula/colors/etc. Customize after creation if needed.
- **No description field** — descriptions can be added later on individual cards. The goal is fast bulk creation, not polished output.
- **No dry-run preview** — the discovery list with thumbnails is the preview; the create button label shows exactly how many cards will land.

### Test Suite Expansion

Added **64 new unit tests** for the `BulkSavedDamageCreator` dialog (`tests/unit/ui/bulk-saved-damage-creator.test.mjs`), covering:

- Image-file filtering and name computation (extension case-insensitivity, relative-path names for recursive scans, leading/trailing slash normalization)
- Form-data reading (defaults, checkbox normalization, type coercion, formula trimming)
- Group-selection parsing (none/existing/new modes, ID validation against the actor, fallbacks for deleted groups)
- Card-data construction (full field mapping, non-mutation of the base template, `groupId` stamping)
- Formula validation (all three formula paths, validator option forwarding)
- Folder scanning (flat, recursive, depth cap, browse errors, source-aware queries)
- Group creation (name handling, sort computation, preservation of existing groups)
- Persistence round-trip (settings load/save, error swallowing, group selection excluded from persistence)

The test suite now stands at **3449 passing tests** (up from 3385 in v14.11.0).

## Migration Notes

No migration required. The new feature is purely additive — no existing fields, items, or settings are changed. The new `bulkSavedDamageCreatorConfig` client setting is registered automatically on first load with a sensible default and is hidden from the settings UI.
