# v14.9.1 — Default "This Character" Toggle & Sync Label Clarity

**Release Date**: July 2026

## Major Features

### Configurable Item Selector Default Scopes

The item selector source scope toggles now default to only **This Character** being enabled, making the most common search case the default. A new **Default Item Selector Scopes** system setting (GM-only) allows configuring which additional scopes (All Characters, Compendiums, World Items) are enabled by default as a comma-separated list.

**What's New**

- **Smart Defaults** — When opening an item sheet on an owned item, only the This Character toggle is active by default. This reduces search noise for the most common workflow (finding items on the current character).
- **Configurable Additional Scopes** — GMs can set `actor, compendium, world` in the new system setting to restore the previous all-enabled behavior, or choose any subset.
- **Per-Item Persistence** — Scope preferences saved on individual items continue to take priority over the system default, so per-item overrides are unaffected.

### Multi-Image Selector Sync Label Clarity

The sync toggles in the Multi-Image Selector popup have been renamed for unambiguous meaning:

- **"Sync Image"** is now **"Sync Image With Action"**
- **"Sync Colors"** is now **"Sync Color With Action"**

These labels make it explicit that the toggle syncs the row's image/colors from the action card's own values, reducing confusion about what "Sync" refers to in context.

No migration required. All changes are additive or behavior-only — existing per-item scope preferences are preserved and take priority over the new system default. The new setting defaults to an empty string (only This Character enabled), maintaining the previous behavior for worlds that never adjust it.
