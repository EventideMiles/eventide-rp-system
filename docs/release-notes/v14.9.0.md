# v14.9.0 — Textless Status Messages & Multi-Image Selector Fixes

**Release Date**: June 2026

## Major Features

### Textless Status Messages

Status items without a description now generate chat messages when applied or removed, matching the behavior of described statuses. Previously, a status with an empty description field was silently skipped during message creation.

**What's New**

- **Apply & Remove Messages** — `createStatusMessage` and `createDeleteStatusMessage` are now invoked for all status items regardless of whether `system.description` is populated. The `item.type === "status" && item.system.description` guard was removed from both the creation and deletion hooks in `chat-listeners.mjs`.
- **User-Requested** — Resolves a long-standing request from GMs who relied on textless statuses (icon-only effects) but still wanted visibility in the chat log when they were toggled.

## Bug Fixes

- **Sheet Image Hover Glow Clipping** — The profile image hover glow on actor and item sheets was being clipped at the header edges. Reduced the profile hover scale from 1.05 to 1.03 to minimize overflow, and added padding/margin breathing room around the actor header image grid cell (4px padding), the item header profile image (4px margin), and the create-buttons container (8px right padding so the `+Template` button is no longer flush against the cell edge).
- **Inflated Action-Card Embedded Item Count** — `_fixActionCardEmbeddedArrays` previously returned `Object.keys(updateData).length` as `itemsFixed`, which could report more than one fixed item for a single update payload and produce an inflated count in migration summaries. It now returns a binary `1`/`0` per item, so the migration report reflects the actual number of items touched.
- **Multi-Image Selector Nested Effect Image Persistence** — Foundry V14's diff machinery was stripping deeply nested `effects[].img` changes inside `ObjectField` values when the Multi-Image Selector saved, so nested effect images appeared to save but reverted on reload. Added `diff: false` to `actionCard.update()` in `MultiImageSelector#onSubmit` to bypass the diff and persist nested image changes verbatim. The deprecated `icon` property sync was also removed from `embedded-item-sheet.mjs`.

## Code Quality

- **Shared Hover-Scale Token** — Replaced the local `$profile-hover-scale` declaration with the existing `tokens.$sheet-profile-hover-scale` (1.02) to keep the hover scale synchronized across all sheets and prevent future drift.
- **Embedded Image Migration v2 Rewrite** — Rewrote `embedded-image-migration.mjs` (v2) to use `diff: false` on all `item.update()` calls, ensuring any pre-existing effect-image desyncs from earlier versions are force-persisted on upgrade rather than silently re-diffed away.

## Migration Notes

This release re-runs the embedded-image migration (v2) with `diff: false`, which will force-persist any nested `effects[].img` values that were previously dropped by Foundry V14's diff machinery. Users who observed multi-image selector changes reverting after reload on v14.8.0 will have their effect images corrected automatically on first load after upgrading; no manual action is required. All other changes are additive or behavior-only and are fully backward-compatible with existing actors, items, and saves.
