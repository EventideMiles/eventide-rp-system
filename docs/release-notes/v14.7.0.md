# v14.7.0 — Action Card Preset Library, NPC Quick Generator & More

**Release Date**: May 2026

## Major Features

### Action Card Preset Library & Creation Overhaul

A comprehensive overhaul of action card creation to eliminate the "blank sheet" problem and tab-switching friction.

**What's New**

- **Preset Library** — 10 pre-built action card templates (Basic Attack, Spell with Status, Healing Spell, Multi-Hit Attack, Defensive Buff, Transformation Trigger, Crit Fishing, AOE Damage, and more) accessible via "New from Template" context menu or `+Template` button in the action cards tab header. Each preset pre-fills all 45+ fields so the card works immediately.
- **Embedded Item Quick-Create** — When selecting an `attackChain` preset, the dialog shows an optional "Create Embedded Combat Power" section that lets users configure power name, roll type, ability, and cost inline — no need to switch to the Embedded Items tab.
- **Smart Defaults on Drag-Drop** — Drag-dropping a combat power now sets `damageCondition` and `statusCondition` to `"oneSuccess"` instead of `"never"`, so newly-created cards actually do something. Combat powers carry their `roll.ability` to the card's stat fields. Gear items auto-set `attemptInventoryReduction: true`.
- **Collapsible Intensify Section** — The 18-field intensify grid is now hidden behind a "Modify Intensification Values" toggle (collapsed by default), with "Set All Target" and "Set All Self" quick controls that batch-set all 9 values at once. Auto-expands for cards with non-default values.
- **Contextual Hints** — Action-oriented hint text added to 7 key fields on the attributes tab explaining gameplay consequences: Mode, First Stat, Second Stat, Damage Condition, Status Condition, Self-Effects Condition, and Transformation Condition.
- **Contextual Callouts** — Three callout boxes on the action card sheet warn about common configuration mistakes: no embedded item configured, all conditions set to "never" (card does nothing), and saved damage cards with no effects. Callouts dismiss automatically when the underlying issue is resolved.
- **Action Card Duplication** — Right-click "Duplicate" on any action card row creates a deep clone including all embedded items, with "(Copy)" appended to the name. Also supports duplicating entire action card groups.
- **Condition Selectors Partial** — The 4 identical 12-option condition dropdowns were extracted into a reusable Handlebars partial (`components/condition-select`), removing ~150 lines of duplicated template code.

### NPC Quick Generator

A dialog that lets GMs create or populate NPCs from a Challenge Rating, dramatically speeding up encounter prep.

**What's New**

- **CR-Based Stat Distribution** — Enter a CR/Level, and the generator evaluates the existing `statPointsFormula` to derive a point budget, then distributes across 6 built-in archetypes (Balanced, Warrior, Spellcaster, Skirmisher, Brute, Defender) using weighted allocation.
- **Archetype & Compendium Support** — Choose from built-in archetypes or select a compendium template via the new `npcTemplateCompendium` setting. Template mode shows a read-only preview; Custom mode provides range sliders with a live budget counter.
- **Stat Overrides** — Optional resolve/power override fields create an "NPC Overrides" feature item with ActiveEffect overrides.
- **Access Points** — Actor directory sidebar context menu (wand icon, GM-only, NPC actors), NPC sheet window control button, or `erps.macros.NpcQuickGenerator`.
- **Token Sync Refactor** — Auto Token Update was split into two independent per-actor toggles:
  - **Auto Token Update** — Syncs prototype token name and image on actor changes
  - **Sync Placed Tokens** — Propagates name/image changes to all placed tokens across scenes
  - Image→token sync moved from the FilePicker callback to `preUpdateActor`/`updateActor` hooks for cleaner separation

### Rest / Recovery Flow

Connects the existing restore target data model to a one-click UI on the actor sheet.

**What's New**

- **Rest Button** — A campground icon button appears next to the character name on actor sheets for users who can edit the sheet. Clicking opens the restore dialog pre-bound to that actor, no token selection required.
- **Permission Relaxation** — The `restore()` method now allows actor owners (not just GMs) to trigger restoration. The standalone `RestoreTarget` macro still requires GM + token targeting.
- **Contextual Titles** — The dialog shows "Rest / Recover" when opened from the actor sheet vs. "Restore Target" from the macro.

### GM Notes Field

A private, GM-only notes field on all actor sheets with full rich text support.

**What's New**

- **Schema** — `gmNotes` StringField on the base actor schema, available on both PCs and NPCs.
- **UI** — Conditionally rendered on the biography tab via `{{#if isGM}}` with a ProseMirror editor in edit mode and enriched HTML display in read-only mode. Invisible to non-GM players.
- **Styling** — Themed biography section header with consistent typography and icon accent.

### Character Summary Chat Command

A one-click way to share or review character state.

**What's New**

- **Summary Card** — A subdued grey card in the actor's status bar (always visible to GMs; opt-in for non-GMs via `enablePostSummary` setting). Clicking posts a detailed character overview to a GM-only whisper.
- **Content** — Name, level, resolve/power (current/max), all abilities with base→total deltas showing status effect influence, active status effect count, and transformation status.

### Gear Equip All / Unequip All

Bulk gear management for the "suit up" and "loot/rest" workflows.

**What's New**

- **Toolbar Buttons** — Equip All and Unequip All buttons in the gear tab toolbar above each gear table.
- **Smart Skipping** — Unequip All skips cursed items (GM-only unlock). Equip All skips items with quantity 0 and items flagged with `skipBulkEquip`.
- **Per-Item Exclude Toggle** — `skipBulkEquip` flag togglable via a row button (`fa-link`/`fa-link-slash` in the controls column) on unequipped gear items. Also accessible via right-click context menu.
- **Summary Chat Card** — Produces a single summary message instead of N individual equip/unequip messages, respecting `showGearEquipMessages` setting.

### Actor Roll History

Per-actor roll tracking for audits and pattern analysis.

**What's New**

- **Automatic Tracking** — Captures native system rolls (ability checks, gear, combat powers, features, damage, heal, initiative) into a per-actor flag array capped at 50 entries (~5 KB). `/r` commands and third-party macros are invisible.
- **Header Button** — Dice icon button on the actor sheet opens the roll history dialog for that actor.
- **Standalone Macro** — `erps.macros.RollHistory.fromSelectedToken()` supports use from the macro bar. Also accessible via right-clicking any actor in the Actor Directory sidebar.
- **Dialog** — Theme-managed 520px dialog shows rolls newest-first with second-granularity relative timestamps and roll type labels mapped from the actual ability used.

### Shared Condition Evaluator

Extracted duplicated condition evaluation logic into a shared utility.

**What's New**

- **`ConditionEvaluator` Utility** — Consolidates the near-identical 12-condition switch statements from `item-action-card-execution.mjs` and `damage-processor.mjs` into a single class. Adding a new condition type now requires a 1-place change instead of 3.

### Code Quality Improvements

- **Formula Validation Dedup** — Extracted `registerFormulaSetting()` helper, removing ~113 lines of identical validation logic across 5 formula settings.
- **JSDoc on Core Public APIs** — Comprehensive documentation for `globalThis.erps` API surface and full IntelliSense support in `types/eventide-rp-system.d.ts`.

## Bug Fixes

- **Unequipped gear context menu** — Fixed a crash where right-clicking an unequipped gear item would throw `TypeError: key.split is not a function` due to a function-based label being passed to Foundry V14's `Localization.localize()`.

## Migration Notes

No data migration is required for this release. All new fields (`gmNotes`, `skipBulkEquip`, `rollHistory`, `selectorScopes`) use nullable or default-empty values. The `restore()` permission relaxation from GM-only to GM-or-owner is backward-compatible. Existing actors, items, and saves are fully compatible.
