# v14.6.0 - Selective Global Item Selector

**Release Date**: May 2026

## Major Features

### Source Scope Filtering for Item Selectors

Item selector combo boxes now include a shared scope filter bar that lets users control which item sources are searched. The filter uses the system's existing toggle component in a compact inline layout with vertical dividers.

**What It Does**

A row of toggle switches appears at the top of each item selector tab when the sheet is editable. Each toggle corresponds to an item source scope:

| Scope | Description |
| ----- | ----------- |
| This Character | Items on the sheet's owning actor (only available for owned items) |
| All Characters | Items across all actor sheets the user can view |
| Compendiums | Items from installed compendium packs |
| World Items | Items in the game world directory |

When toggled off, items from that source are excluded from the search results. All combo boxes on the same item sheet share the same scope selection — toggling a scope on one selector instantly updates all others.

**How It Works**

1. Open an item sheet with an editable combo box (action items, effects, transformations, etc.)
2. A **Filter by source** bar appears at the top of the tab with toggle switches for each available scope
3. Toggle scopes on or off to include or exclude item sources
4. The combo box dropdown immediately refreshes to show only items from enabled scopes
5. Scope preferences are automatically saved to the item and persist across sessions

**Use Cases**

- **Narrow search to owned items** — Toggle off Compendiums and World Items to see only items already on the character's sheet
- **Compendium-only browsing** — Toggle off everything except Compendiums to quickly find items from game packs
- **Exclude character items** — Toggle off All Characters and This Character to search only world and compendium sources
- **Default behavior preserved** — With all scopes enabled (the default), the search works exactly as before

**Behavior Details**

| Scenario | This Character | All Characters | Compendiums | World Items | Result |
| -------- | -------------- | -------------- | ----------- | ----------- | ------ |
| Default (all on) | On | On | On | On | All items visible (unchanged behavior) |
| Owned item, this character only | On | Off | Off | Off | Only items on the owning actor |
| Unowned item, no this character | N/A | On | On | On | All scopes except this character |
| All toggled off | — | — | — | — | All scopes re-enabled (prevents empty results) |

**Important Notes**

- The **This Character** scope only appears when the item is owned by an actor (embedded items). For unowned world items, it is automatically hidden
- Scope preferences are saved via `item.update()` with `render: false` to prevent the sheet from re-rendering and losing UI state
- If all toggles are unchecked, all scopes are automatically re-enabled to prevent returning zero search results
- Deduplication priority ensures that when the same item appears in multiple sources, the closest source takes precedence: This Character > All Characters > World Items > Compendiums

**Schema Changes**

The item data model now includes one new nullable field:

| Field | Default | Description |
| ----- | ------- | ----------- |
| `system.selectorScopes` | `null` | Array of enabled scope identifiers, or `null` for all scopes enabled |

## Migration Notes

No migration is required for this release. The `selectorScopes` field defaults to `null`, which enables all scopes and preserves existing search behavior. Existing saves and data are fully compatible.
