# v14.11.0 — Combat Power Linking

**Release Date**: July 2026

## Major Features

### Combat Power Linking

Action cards can now **link to real combat power items** on the actor instead of storing duplicate copies. This enables a single-source-of-truth workflow: edit the combat power once, and all action cards that reference it update automatically.

**What's New**

- **New `embeddedItemRef` field** — Action cards that reference a combat power on the actor store its item ID. When set, `getEmbeddedItem` resolves the real item directly from the actor's inventory, with the `embeddedItem` snapshot kept as a cache/fallback.
- **Automatic linking** — When you drag a combat power from an actor onto an action card, the card automatically links to it (stores the reference) instead of creating a duplicate.
- **Three new methods on action cards**:
  - `linkEmbeddedItem(itemId)` — Links the card to an existing combat power on the actor
  - `unlinkEmbeddedItem()` — Breaks the link (retains the snapshot)
  - `syncFromSource()` — Refreshes the snapshot from the linked source item
- **Smart card routing preserved** — Linked items use the same resolution path as embedded items, so all existing card display, execution, and power-damage routing works unchanged.
- **Snapshot fallback** — If a linked item is deleted from the actor, the action card falls back to the `embeddedItem` snapshot, ensuring no data loss.

### Migration

On first load after updating, the system automatically migrates existing action cards:

1. **Scans** all action cards on all actors for embedded combat powers without a reference.
2. **Deduplicates** — If two action cards embed identical combat powers (same name, image, cost, roll formula), a single combat power item is created on the actor and both cards link to it.
3. **Creates** — For unique embedded combat powers, a new combat power item is created on the actor from the snapshot.
4. **Links** — Sets `embeddedItemRef` on each migrated action card.

The migration is tracked via `migrationVersion` (set to `3`) and only runs once.

## Version Information

## Migration Notes

- The combat power linking migration runs automatically on first load.
- Existing action cards retain their `embeddedItem` snapshot as a fallback.
- Gear and feature embedded items are NOT affected — only combat powers are linked.
- The migration is non-destructive: if it fails for any card, that card continues to work with its embedded snapshot.
