# v14.8.0 — Multi-Image Selector for Action Cards

**Release Date**: May 2026

## Major Features

### Multi-Image Selector Popup

A dedicated popup application (`MultiImageSelector`) that provides a unified interface for managing images and colors across all visual components of an action card, replacing the single-image FilePicker for standalone action cards (saved-damage-mode cards continue to use the normal FilePicker).

**What's New**

- **5 Sections** — Organizes visual customization into logical groups:
  - **Action Card** — Main card image, background color, and text color
  - **Combat Power** — Embedded combat power image and colors, with Sync Image / Sync Colors toggles. "Add Combat Power" button creates a blank embedded power defaulting to the action card's current image and colors. Hidden entirely for saved-damage-mode cards.
  - **Status Effects** — Per-effect image and color controls with independent sync toggles. "Add Status Effect" button creates a blank embedded status. Hidden if there are no status effects to show.
  - **Transformations** — Per-transformation image and color controls with sync toggles. Shown only when transformations exist.
  - **Self-Effects** — Per-self-effect image and color controls with independent sync toggles. "Add Self Effect" button creates a blank embedded self-effect.

- **Live Sync Toggles** — Checkboxes for "Sync Image" and "Sync Colors" that immediately copy the action card's current image, background color, and/or text color to the target row. When a sync toggle is active, the corresponding inputs become disabled (non-interactive with visual indicator). Changes to the action card's own image or colors automatically propagate in real time to any rows with active sync toggles. Unchecking restores the previous value.

- **FilePicker Integration** — Clicking any thumbnail image throughout the popup opens Foundry's FilePicker for image selection, with coordinates relative to the popup position.

- **Trigger Points** — Two access paths:
  - **Item Sheet Header** — Clicking the profile image on an action card's sheet (non-saved-damage mode) opens the Multi-Image Selector instead of the normal FilePicker
  - **Actor Sheet Context Menu** — Right-clicking any action card row in the actor sheet shows an "Edit Images" entry (fa-images icon)

- **Form Data Preservation** — Unsaved form data (including scroll position) is preserved across re-renders when adding new embedded items, preventing loss of in-progress edits.

- **"Original Power Image" Label** — The popup header displays a contextual label identifying the action card image as the source for sync operations.

### Code Quality & Cleanup

- **Removed Obsolete Callout** — The "saved damage with no effects" callout was removed from the item sheet (`item-sheet.mjs`), along with its translation key (`SavedDamageNoEffects`)
- **ESLint Global** — `FormData` added as a readonly global for the new module
