# v14.0.0 - Foundry VTT v14 Compatibility & ActiveEffect Schema Migration

**Release Date**: April 2026

## ⚠️ Breaking Changes

This is a **major version release** requiring **Foundry VTT v14 or later**. The system is no longer compatible with Foundry VTT v13 due to significant changes in the ActiveEffect data model introduced by Foundry VTT v14.

**Upgrade Requirements**

- Foundry VTT v14 or later is required
- The system will automatically migrate existing ActiveEffect data on first launch
- Migration is one-way; **back up your world before upgrading**
- Allow the migration to complete fully before modifying actors or items

## Major Features

### Foundry VTT v14 ActiveEffect Schema Migration

A comprehensive migration system that updates all ActiveEffect data to the new Foundry VTT v14 schema.

**What Changed in the ActiveEffect Schema?**

Foundry VTT v14 introduced significant structural changes to the ActiveEffect data model. This system update ensures all existing effects are properly migrated and all new effects conform to the v14 schema.

**Schema Changes**

| Field            | V13                                              | V14                                                           |
| ---------------- | ------------------------------------------------ | ------------------------------------------------------------- |
| Image            | `icon`                                           | `img` (renamed)                                               |
| Effect Type      | _(absent)_                                       | `type` (new; defaults to `"base"`)                            |
| Changes Location | `changes` (top-level)                            | `system.changes` (nested under system)                        |
| Change Phase     | _(absent)_                                       | `phase` (new; defaults to `"initial"`)                        |
| Change Mode      | Numeric mode (e.g., `5` for override)            | String `type` (e.g., `"override"`)                            |
| Start Fields     | `startRound`, `startTurn`, `startTime`           | `start: { combat, combatant, initiative, round, time, turn }` |
| Statuses         | `Set`                                            | `Array`                                                       |
| Show Icon        | _(absent: used duration to handle icon showing)_ | `0` = NEVER, `2` = ALWAYS                                     |
| Duration         | Flat structure with `seconds`, `rounds`, etc.    | Restructured with `expired`, `expiry`, `units`, `value`       |
| Origin           | Empty string allowed                             | `null` (empty strings migrated to null)                       |

**Migration System**

The [`V14ActiveEffectMigration`](../../module/services/migrations/v14-active-effect-migration.mjs) class handles the complete migration process:

- **Automatic Migration**: Runs on first launch after upgrade, processing all actors, world items, and compendium items
- **Version Tracking**: Uses `v14MigrationVersion` setting to track migration status (current: `14.0.1`)
- **GM-Only Execution**: Migration only runs for GM users to prevent conflicts
- **Error Handling**: Failures are logged and re-thrown; migration version is only updated on success
- **Compendium Support**: Migrates effects inside compendium packs, not just world documents

**showIcon Migration (Always-On Fix)**

The `showIcon=1` (CONDITIONAL/HOVER) fix runs on **every startup**, regardless of migration version, ensuring previously-migrated worlds also get corrected. This behavior may change in future versions of the system:

- `showIcon === 1` → `ALWAYS` (2) if the effect had a duration, `NEVER` (0) if permanent
- `showIcon === undefined` or `null` → Duration-based logic applied
- Item type awareness: `actionCard`, `combatPower`, and `transformation` effects always get `NEVER` (0); `status`, `feature`, and `gear` effects are user-controllable

**Benefits**

- Seamless upgrade from Foundry VTT v13 to v14
- All existing game data preserved through automatic migration
- No manual steps required for the migration itself
- Continuous correction of edge-case misconfigurations (showIcon fix)

### ActiveEffect Creation Overhaul

All code that creates ActiveEffects throughout the system has been updated to conform to the v14 schema.

**What Changed?**

Every place in the codebase that creates or manages ActiveEffects now uses the v14 fields:

- New effects use `img` instead of `icon`
- All new effects include `type: "base"`
- Effect changes are nested under `system.changes`
- Change mode uses string `type` instead of numeric mode
- `showIcon` replaces duration-based visibility toggling
- Duration uses the new v14 schema structure
- New effects include proper `start` object with all nested fields

**Where This Applies**

- [`ItemActionCardMixin`](../../module/documents/mixins/item-action-card.mjs) — embedded item and status effect creation in action cards
- [`CharacterEffects`](../../module/helpers/character-effects.mjs) — reads `system.changes` instead of top-level `changes`; uses string `type` instead of numeric mode
- [`StatusIntensification`](../../module/helpers/status-intensification.mjs) — reads/writes `system.changes` for effect updates
- [`EmbeddedItemSheet`](../../module/ui/components/creator-application.mjs) — effect display toggle now uses `showIcon` instead of duration manipulation
- [`EffectManager`](../../module/services/effect-manager.mjs) — all effect creation uses v14 schema

### Effect Display Toggle Refactored

The embedded item sheet's effect display toggle has been completely refactored.

**Before (V13)**

Effects were shown/hidden by manipulating the `duration` field — setting `seconds: 604800` to show, `seconds: 0` to hide. This was a workaround that relied on duration-based visibility.

**After (V14)**

The toggle now uses Foundry VTT v14's proper `showIcon` property:

- **Show**: `showIcon = CONST.ACTIVE_EFFECT_SHOW_ICON.ALWAYS`
- **Hide**: `showIcon = CONST.ACTIVE_EFFECT_SHOW_ICON.NEVER`

This is cleaner, more performant, and semantically correct — no longer abusing duration to control visibility.

### Damage Condition Default: oneSuccess

**Change**: The default `damageCondition` for attack chains has been changed to `oneSuccess`.

**Why?** Previously the default could lead to inconsistent behavior where damage would not apply when players expected it to. The `oneSuccess` condition is more intuitive and consistent — if at least one attack in a chain hits, damage applies. This aligns better with user expectations and makes the system feel more responsive.

**Impact**: New action cards default to `oneSuccess`. Existing action cards are not modified.

## Code Quality & Infrastructure

### Code Consolidation & Cleanup

Significant code cleanup and consolidation across the codebase as part of the v14 preparation and review:

- **Theme Management**: Moved popup theme management to a shared mixin, eliminating duplicated code across multiple popup types

## Upgrade Guide

### For Players

**Before Upgrading**

1. Coordinate with your GM — this is a major version update
2. Ensure your world is backed up
3. Confirm your Foundry VTT server is updated to v14

**After Upgrading**

- The migration runs automatically when a GM first opens the world
- Do not interrupt the migration process
- All your characters, action cards, status effects, and transformations will be preserved
- Effect visibility may change for some items — check your action cards and status effects if something looks different
- New action cards default to `oneSuccess` damage condition for more consistent behavior

### For Game Masters

**Migration Details**

- The migration is automatic and requires no manual intervention
- A console notification will indicate when migration starts and completes
- If migration fails, check the browser console (F12) for error details
- The `v14MigrationVersion` setting tracks migration completion
- The `showIcon=1` fix runs on every startup as a safety measure

**Post-Upgrade Verification**

- Review compendium packs for any effects that may need manual adjustment
- Check active effects on important NPCs and creatures
- Verify that effect visibility (show/hide icon) matches expectations
- Test action card execution to confirm effects apply correctly
- If issues arise, check the browser console for migration-related warnings

**Testing Infrastructure Note**

Automated testing is currently disabled due to `@rayners/foundry-test-utils` incompatibility with Foundry VTT v14. The test suite will be re-enabled once the testing utility library is updated for v14 compatibility.
