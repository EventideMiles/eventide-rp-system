# v14.3.0 - Action Card Self-Effects

**Release Date**: April 2026

## Major Features

### Action Card Self-Effects

Action cards now support a **Self-Effects** system that allows configuring status, gear, and feature items to apply to the card owner (rather than the target) based on a configurable degree-of-success condition. This enables mechanics like drain attacks, double-edged attacks, recoil effects, and self-buffing abilities.

**What It Does**

Self-effects are embedded items stored on the action card that are automatically applied to the character using the card when the configured condition is met. This allows for:

- **Drain attacks** — Apply a debuff to yourself when attacking (e.g., exhaustion from powerful strikes)
- **Double-edged attacks** — Self-inflict status effects as a trade-off for high damage
- **Recoil effects** — Apply gear depletion or penalties to the user
- **Self-buffing** — Grant yourself a buff when an attack succeeds

**How It Works**

1. Open an action card and navigate to the **Embedded Items** tab
2. In the new **Self-Effects** section, drop status, gear, or feature items
3. Configure when self-effects should apply using the **Self-Effect Condition** selector
4. Optionally set a **Self-Effect Application Limit** on the Config tab to control how many times effects apply per execution
5. When the action card is executed, the system checks the condition against the roll result and applies matching effects to the card owner

**Condition Options**

Self-effects can be configured to apply based on the following conditions:

| Condition | Description |
| --------- | ----------- |
| `oneSuccess` (default) | At least one success on the roll |
| `twoSuccesses` | Two or more successes |
| `zeroSuccesses` | No successes (complete failure) |
| `criticalSuccess` | Natural 20 on any die |
| `criticalFailure` | Natural 1 on any die |
| `rollValue` | Total meets or exceeds threshold |
| `rollUnderValue` | Total is at or below threshold |
| `rollOnValue` | Total exactly equals threshold |
| `rollEven` | Total is an even number |
| `rollOdd` | Total is an odd number |
| `always` | Always applies regardless of roll |
| `never` | Never applies (effectively disabled) |

**Application Limit**

The **Self-Effect Application Limit** (found on the Config tab) controls how many times self-effects can apply per action card execution. This is especially useful for multi-attack action cards:

- **Default: 1** — Self-effects apply once per execution, even if the card hits multiple targets
- **Set to 0** — Unlimited applications (self-effects apply for every target hit)

**Schema Changes**

The action card data model now includes:

| Field | Default | Description |
| ----- | ------- | ----------- |
| `embeddedSelfEffects` | `[]` | Array of embedded status, gear, and feature items |
| `selfEffectsConfig.condition` | `"oneSuccess"` | When self-effects should apply |
| `selfEffectsConfig.threshold` | `0` | Threshold value for roll value conditions |
| `selfEffectsApplicationLimit` | `1` | Max times self-effects apply per execution (0 = unlimited) |

### Self-Effects UI

A new **Self-Effects** panel has been added to the Embedded Items tab on action card sheets. The panel provides:

- **Drag-and-drop zone** for dropping status, gear, and feature items from the Items directory
- **Item selector combo box** for searching and selecting items by name
- **Edit and remove controls** for managing embedded self-effects
- **Create New Self-Effect button** for creating new items inline

The **Attributes** tab includes a new Self-Effect Configuration section with condition selector and threshold input, and the **Config** tab includes the application limit setting.

### Status Intensification Integration

Self-effects that are status items use the existing `StatusIntensification.applyOrIntensifyStatus()` system. This means if a character already has the same status, it will be intensified (stacked) rather than creating a duplicate. This provides consistent behavior with target-applied status effects.

## Migration Notes

No migration is required for this release. Self-effects are a new addition to the action card data model and will default to an empty array with `oneSuccess` condition for all existing action cards. Existing saves and data are fully compatible.
