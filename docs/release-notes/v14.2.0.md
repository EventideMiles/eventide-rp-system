# v14.2.0 - Healing Increase with Multiplicative Support

**Release Date**: April 2026

## Major Features

### Healing Increase Hidden Ability

A new hidden ability `healIncrease` has been added that modifies the amount of healing a character receives. This ability works like other hidden abilities (e.g., `dice`, `criticalMax`) and supports both additive and multiplicative effects.

**What It Does**

The `healIncrease` ability applies a modifier to healing rolls before the healing is applied to the character's resolve. This allows for:

- **Passive healing bonuses** â€” Characters can have permanent healing increases from features or status effects
- **Multiplicative healing scaling** â€” Use Multiply, Divide, Multiply (Buff), or Multiply (Debuff) modes to scale healing proportionally
- **Dynamic healing adjustments** â€” Healing can be enhanced or reduced based on character state, conditions, or items

**How It Works**

1. A healing action is executed (e.g., a healing combat power or action card)
2. The healing roll is calculated normally
3. The `healIncrease` ability total is calculated (including all additive and multiplicative effects)
4. The healing total is increased by `healIncrease`
5. The final healing is applied to the character's resolve, with a minimum of 1

**Minimum Healing Constraint**

To prevent healing from turning into damage (which would happen if `healIncrease` is negative enough), the system ensures a minimum of 1 healing is always applied. Even if the calculation would result in 0 or negative healing, the character will still receive 1 point of healing.

**Example Scenario**

1. Your character rolls 6 points of healing
2. You have a status effect that adds +2 to healing (Add mode on `healIncrease`)
3. Another effect doubles healing (Multiply Ă—2 on `healIncrease`)
4. Final calculation: 6 + 2 = 8 (base + add), then 8 Ă— 2 = 16 (final healing)
5. Character receives 16 points of healing (or overheals if at max resolve)

**Multiplicative Modes**

The `healIncrease` ability supports all four multiplicative modes introduced in v14.1.0:

- **Multiply** â€” Neutral multiplication (e.g., Ă—2 doubles all healing increases)
- **Divide** â€” Neutral division (e.g., Ă·2 halves all healing increases)
- **Multiply (Buff)** â€” Multiply positive increases, divide negative increases (always beneficial)
- **Multiply (Debuff)** â€” Divide positive increases, multiply negative increases (always harmful)

**Schema Changes**

The `healIncrease` ability includes the same schema fields as other hidden abilities:

| Field | Default | Description |
| ------- | --------- | ----------- |
| `base` | 0 | Base value set on the character |
| `add` | 0 | Compounded by Add effects |
| `override` | null | Overrides all other calculations if set |
| `transformOverride` | null | Applied after base but before multiplicative effects |
| `multiplyNeutral` | 1 | Compounded by Multiply effects |
| `divideNeutral` | 1 | Compounded by Divide effects |
| `multiplyBuff` | 1 | Compounded by Multiply (Buff) effects |
| `multiplyDebuff` | 1 | Compounded by Multiply (Debuff) effects |

### Overhealing Display

Chat messages now clearly display when healing exceeds a character's maximum resolve (overhealing). This provides transparency to players and GMs about how much healing was "wasted" because the character was already at or near full health.

**What Changed**

Previously, when healing would push a character beyond their maximum resolve, the excess was silently discarded. Now, the chat card explicitly shows:

- **Total healing rolled** â€” The full amount of healing calculated
- **Healing applied** â€” The amount that actually increased resolve (up to max)
- **Overhealing amount** â€” The excess healing that couldn't be applied

**Example Display**

```text
Healing: 12
Applied: 5 (Max Resolve: 20/20)
Overhealing: 7
```

This helps players make informed decisions about when to use healing resources and allows GMs to track healing efficiency.

### Formula Construction for Healing

The healing calculation formula has been updated to properly construct and display the order of operations when both additive and multiplicative healing effects are present.

**How the Formula is Built**

The healing formula now follows the same calculation order as other abilities with multiplicative support:

1. **Base healing roll** â€” The actual dice roll for healing
2. **Additive effects** â€” All Add, Override, and TransformChange effects are summed
3. **Neutral Multiply** â€” Multiply effects compound the total
4. **Neutral Divide** â€” Divide effects compound the total
5. **Multiply (Buff)** â€” Positive totals are multiplied, negative totals are divided
6. **Multiply (Debuff)** â€” Positive totals are divided, negative totals are multiplied

**Formula Display Example**

When a character has multiple healing modifiers, the formula displayed in chat shows the complete calculation:

```text
Formula: (6 + 2) Ă— 2 = 16
```

This clarity helps players understand exactly how their healing is being calculated.

### Formula Display Bug Fix

**Issue**: Roll terms (dice results) were not appearing in nested math formulas, making it difficult for players to see what they actually rolled.

**What Was Broken**

When a formula contained nested math operations with roll terms (e.g., `(@roll.total + @healIncrease.total)`), the roll terms would evaluate to their numeric values but wouldn't display the actual dice roll that produced them. This made troubleshooting and verification of results difficult.

**What Was Fixed**

The formula display system now properly extracts and shows roll terms within nested math expressions. Players can now see:

- The actual dice roll (e.g., `3d6 = 12`)
- How that roll was used in the larger formula
- The step-by-step calculation

**Example Before and After**

Before:

```text
Healing: (12 + 4) Ă— 1.5 = 24
```

After:

```text
Roll: 3d6 = 12
Formula: (12 + 4) Ă— 1.5 = 24
```

This fix applies to all roll formulas throughout the system, not just healing.
