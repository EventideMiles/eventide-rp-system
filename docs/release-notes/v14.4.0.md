# v14.4.0 - Configurable Intensification & Accessibility Improvements

**Release Date**: May 2026

## Major Features

### Configurable Status Intensification Behavior

Status effects now support configurable intensification rates, allowing different effect types (Add, Advantage, Disadvantage, AC, Multiply, etc.) to increment by different amounts when status effects are reapplied.

**What It Does**

The new `intensifyBehavior` configuration on action cards allows fine-grained control over how much each effect type increments when a status effect is intensified:

- **Add** — Increment amount for additive effects (default: 1)
- **Advantage** — Increment amount for advantage dice (default: 1)
- **Disadvantage** — Increment amount for disadvantage dice (default: 1)
- **AC** — Increment amount for armor class changes (default: 1)
- **Multiply** — Increment amount for neutral multiply effects (default: 1)
- **Divide** — Increment amount for divide effects (default: 1)
- **Multiply Buff** — Increment amount for beneficial multiplicative effects (default: 1)
- **Multiply Debuff** — Increment amount for harmful multiplicative effects (default: 1)
- **Override** — Increment amount for override effects (default: 1)

**How It Works**

1. Action cards now have an `intensifyBehavior` configuration with separate settings for **target effects** and **self effects**
2. When a status effect is applied to a target that already has it, the system uses the configured increment values
3. Each effect mode type can have its own custom increment amount
4. This allows effects to intensify at different rates (e.g., +2 to AC per stack but only +1 advantage)

**Use Cases**

- **Diminishing returns** — Set lower increment values for powerful effects
- **Accelerating effects** — Set higher increment values for effects that should scale faster
- **Selective scaling** — Only intensify certain effect types (set others to 0)
- **Custom balance** — Fine-tune how different effect modes stack in your game

**UI Integration**

- New Intensify Behavior section on the Config tab of action cards
- Separate fieldsets for Target Effects and Self Effects
- Number inputs for each effect mode type with min value of 0
- Default value of 1 for all increment amounts (standard stacking behavior)

### Effect Mode Display in Chat Cards and Popups

Effect modes (Add, Override, Multiply, etc.) are now correctly displayed in chat cards and popup dialogs.

**What Changed**

- Chat cards now show the correct effect mode for character effects
- Popup dialogs display effect mode information accurately
- Consistent terminology across all UI surfaces

**Benefits**

- Players can clearly see how effects are applied to their characters
- GMs can verify effect configurations before applying
- Reduces confusion about stacking behavior

### V14 MessageMode API Migration

Migrated all deprecated roll mode APIs to Foundry VTT v14's new `messageMode` API.

**What Changed**

The following modules have been updated to use the new V14 API:

- `roll-dice.mjs` — Partial migration for dice rolling
- `actor-sheet.mjs` — Sheet message mode handling
- `actor-rolls.mjs` — Actor roll message configuration
- `system-messages.mjs` — System messaging utilities
- `chat-message-builder.mjs` — Chat message construction

**Benefits**

- Removes deprecation warnings from console
- Ensures compatibility with future Foundry VTT versions
- Cleaner, more maintainable code

## Accessibility Features

### Keyboard-Navigable Popups

Popups are now fully keyboard-navigable, improving accessibility for users who prefer or require keyboard navigation.

**What's New**

- **Tab Navigation**: Move between interactive elements with Tab/Shift+Tab
- **Enter/Space Activation**: Activate buttons and controls with Enter or Space
- **Focus Visible Styles**: Clear visual indication of focused elements
- **Keyboard Context Management**: Proper focus management when popups open/close

**Benefits**

- Improved accessibility for keyboard-only users
- Better user experience for power users
- WCAG compliance improvements

### Accessibility Attributes for Transformation Rows

Transformation rows now include proper accessibility attributes for screen readers.

**What Changed**

- Added ARIA labels to transformation row elements
- Improved semantic structure for better screen reader support
- Enhanced focus management for keyboard navigation

## Bug Fixes

### Gear Subtab Initialization Fix

**Issue**: Gear subtabs were not initializing properly due to a mixin method override conflict.

**Fix**: Resolved the conflict by ensuring proper method resolution order in the mixin chain.

**Impact**: Gear subtabs now initialize correctly on first load.

**Pull Request**: #187

### Action Card Race Condition Fix

**Issue**: A race condition in repetition context handling caused unpredictable behavior in multi-execution action cards.

**Fix**: Added proper async/await handling and context isolation to prevent race conditions.

**Impact**: Multi-execution action cards now reliably track context between repetitions.

**Pull Request**: #182

### Overly Aggressive DOM Cleanup Fix

**Issue**: DOM cleanup on visibilitychange events was breaking sheets in certain scenarios.

**Fix**: Removed aggressive DOM cleanup that was interfering with normal sheet operation.

**Impact**: Sheets remain functional after visibility changes (tab switching, minimizing, etc.).

**Pull Request**: #184

### Popup Button Clickability Fix

**Issue**: Popup buttons were not clickable across their entire region on smaller screens.

**Fix**: Adjusted CSS to ensure buttons are fully clickable regardless of screen size.

**Impact**: All popup buttons now respond to clicks on their entire visible area.

**Pull Request**: #180

### Performance Dashboard Fixes

**Issue**: Performance dashboard template and localization needed updates; theme instances and migration version display were broken.

**Fix**: Updated template structure, fixed theme instance handling, and corrected migration version diagnostics.

**Impact**: Performance dashboard now displays correctly with accurate information.

**Pull Request**: #185
