# v14.1.0 - Multiplicative Character Effects (Multiply & Divide)

**Release Date**: April 2026

## Major Features

### Multiply & Divide Character Effect Modes

Character effects now support multiplicative operations using Foundry VTT v14's new `"multiply"` ActiveEffect change type. This enables effects that scale abilities proportionally rather than just adding or overriding values.

**New Effect Modes**

Four new modes are available in both Regular Effects and Hidden Effects dropdowns:

| Mode | Effect | Example |
| ------ | -------- | --------- |
| **Multiply** | Multiplies the ability total by the value (neutral — works the same on positives and negatives) | Acrobatics 10 × 2 = 20 |
| **Divide** | Divides the ability total by the value (neutral — works the same on positives and negatives) | Acrobatics 10 ÷ 2 = 5 |
| **Multiply (Buff)** | Exclusively beneficial: multiplies positives, divides negatives | Acrobatics -4 × 2 (buff) = -2 |
| **Multiply (Debuff)** | Exclusively harmful: divides positives, multiplies negatives | Acrobatics 10 × 2 (debuff) = 5 |

**Order of Operations**

Multiplicative effects are applied **after** all additive calculations:

1. Base value (or override/transform override if set)
2. Additive changes (change + transformChange)
3. Neutral Multiply (multiply all values the same way)
4. Neutral Divide (divide all values the same way)
5. Multiply (Buff) (multiply positives, divide negatives)
6. Multiply (Debuff) (divide positives, multiply negatives)
7. After each multiply/divide step, intermediate results are rounded to the nearest hundredth to prevent JavaScript floating point accumulation errors
8. Final result is rounded to the nearest integer via `Math.round()`

**How It Works: Neutral Multiply & Divide**

These apply uniformly regardless of whether the total is positive or negative:

- **Multiply**: Acrobatics 10 × 2 = 20; Acrobatics -4 × 2 = -8
- **Divide**: Acrobatics 10 ÷ 2 = 5; Acrobatics -4 ÷ 2 = -2

**How It Works: Exclusively Buff/Debuff**

These modes "flip" their behavior based on the sign of the total, ensuring the effect is always beneficial (Buff) or always harmful (Debuff):

- **Multiply (Buff)** on positive total: multiplies (10 × 2 = 20 — more positive, which is beneficial)
- **Multiply (Buff)** on negative total: divides (-4 ÷ 2 = -2 — less negative, which is beneficial)
- **Multiply (Debuff)** on positive total: divides (10 ÷ 2 = 5 — less positive, which is harmful)
- **Multiply (Debuff)** on negative total: multiplies (-4 × 2 = -8 — more negative, which is harmful)

**Zero handling**: Zero totals remain zero under both Buff and Debuff modes.

**Example Scenario**

1. Your Acrobatics is naturally 10
2. A status effect reduces it by 2 (Add -2) → total is 8
3. Another effect halves it (Multiply 0.5) → total is 4

**Where This Applies**

- All visible abilities (Acrobatics, Physique, Fortitude, Will, Wits)
- All hidden abilities (Dice, Critical Max, Critical Min, Fumble Min, Fumble Max, Vulnerability)
- Power and Resolve multipliers continue to use their existing percentage system and are not affected by these new modes

**Schema Changes**

Each ability (visible and hidden) now includes four new fields:

| Field | Default | Description |
| ------- | --------- | ----------- |
| `multiplyNeutral` | 1 | Compounded by "Multiply" effects |
| `divideNeutral` | 1 | Compounded by "Divide" effects |
| `multiplyBuff` | 1 | Compounded by "Multiply (Buff)" effects |
| `multiplyDebuff` | 1 | Compounded by "Multiply (Debuff)" effects |

A default value of 1 means no modification — the ability total passes through unchanged.

**ActiveEffect Integration**

Multiply and divide effects use Foundry V14's `"multiply"` change type, which compounds multiple effects on the same key. For example, two "Multiply ×2" effects on Acrobatics would compound the `multiplyNeutral` field to 4 (2 × 2), not set it to 2.
